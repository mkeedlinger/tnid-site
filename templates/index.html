Features

  • Human-readable — IDs show their type, no cross-referencing needed
  • Type-safe — Compile-time checking in supported languages
  • Unambiguous — One canonical string per ID (unlike UUID hex)
  • Sortable — Strings sort correctly as-is
  • URL-safe — No percent-encoding needed
  • Time-ordered option — V0 sorts chronologically (like UUIDv7)
  • Random option — V1 for full randomness (like UUIDv4)
  • UUID-compatible — Works with existing tooling and infrastructure
  • Same size — 128 bits, no storage overhead
  • Encryption extension — Hide timestamps by converting V0 to V1
